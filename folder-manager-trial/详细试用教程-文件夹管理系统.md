# 哈利 / Windows 文件夹管理系统详细试用教程

## 1. 这是什么

这是一个本地桌面工具，用来把项目文件夹管理这件事做得更标准、更可检查、更容易交接。

它适合：
- 项目资料越来越乱，需要重新建立目录规则
- 研发 / 设计 / 交付项目需要统一目录结构
- 需要把历史项目整理成可交接状态
- 需要快速搜索、定位、打开高频文件
- 需要对两个文件或两个文件夹做差异对比

---

## 2. 这个工具能帮你做什么

你可以把它理解为 4 个核心能力：

1. **定标准**
   - 用 `File_list.md` 定义标准目录结构

2. **建结构**
   - 根据模板一键创建目录和空文件

3. **查偏差**
   - 扫描真实目录
   - 对比标准结构和当前实际结构
   - 看缺失项、多余项、变更项

4. **提效率**
   - 搜索文件
   - 打开文件
   - 定位文件夹
   - 复制完整路径 / 相对路径
   - 收藏高频文件
   - 对比两个文件 / 文件夹
   - 用 AI 自动总结差异

---

## 3. 第一次试用前的准备

建议先准备一个**测试项目目录**，不要一上来就拿正式生产项目直接操作。

例如：
- `D:\DemoProject`
- `D:\Project-Test`

这样即使你随便试，也不会影响正式资料。

## 4. 启动后先理解两个主界面

这个工具主要有两个工作区：

### 4.1 项目设置
负责：
- 选择目标文件夹
- 选择模板
- 编辑 `File_list.md`
- 生成结构
- 扫描结构
- 保存项目配置
- 补齐缺失项

### 4.2 文件检索与对比
负责：
- 搜索文件
- 打开 / 定位 / 复制路径
- 文件 / 文件夹对比
- 查看结构化结果
- AI 总结差异
- 最近目录 / 收藏文件

你可以简单记成：
- **左边/前一个页面是“搭规则”**
- **右边/后一个页面是“查文件、看差异”**

## 5. 最推荐的新手试用流程

这是最稳的试法：

### 第 1 步：选择目标文件夹
点击“选择文件夹”，选中你要管理的项目目录。

### 第 2 步：选择模板类型
当前内置模板包括：
- 通用模板
- DHF模板
- 硬件模板
- 结构模板

如果你不确定，就先选“通用模板”。

### 第 3 步：点击“加载模板”
模板内容会进入 `File_list.md` 编辑区。

### 第 4 步：看一眼 `File_list.md`
你可以先不改，直接试。
如果要改，记住规则：
- 目录以 `/` 结尾
- 文件不加 `/`
- 每层 2 个空格缩进
- 行尾可以用 `#` 加注释

示例：

```text
ProjectRoot/
  README.md
  docs/
    spec.md
  src/
    main.py
```

### 第 5 步：点击“初始化模板”
会在目标目录下生成：
- `File_list.md`

### 第 6 步：点击“生成结构”
程序会根据模板创建目录和空文件。

### 第 7 步：点击“扫描结构”
程序会生成当前目录树快照，方便后续留档与复核。

### 第 8 步：打开“文件/文件夹对比”
这里是这版比较接近产品的核心能力之一。

你可以：
- 对比两个文件
- 对比两个文件夹
- 看左右差异
- 看颜色高亮
- 用 AI 总结结果

### 第 9 步：必要时点击“一键补齐缺失项”
如果标准模板里定义了、但目录里缺失的内容，可以自动补齐。

## 6. 如何理解对比窗口

这是试用时最值得重点体验的部分。

### 6.1 左右分别代表什么
请记住一句话：

- **左边 = OLD / 基准**
- **右边 = NEW / 当前**

也就是：
- 左边通常是你拿来当标准、当旧版本、当参考的一侧
- 右边通常是当前版本、实际版本、变化结果的一侧

### 6.2 文件对比时怎么看
文件对比时：
- 左侧看旧版本内容
- 右侧看新版本内容和 diff 结果

### 6.3 文件夹对比时怎么看
文件夹对比时：
- 左侧看“仅左侧存在”与“一致文件”
- 右侧看“仅右侧存在”与“变更文件”

### 6.4 颜色代表什么
- **绿色**：新增
- **红色**：删除
- **蓝色**：文件头 / 路径信息
- **紫色**：变更块定位

如果你想快速看：
1. 先看右侧
2. 再看下方 AI 总结

## 7. 文件搜索怎么试

进入“文件检索与对比”页后，你可以：

### 搜索文件
在搜索框里输入：
- 文件名
- 路径片段
- 多个关键词

例如：
- `readme`
- `docs spec`
- `bom`

### 搜索结果能做什么
选中某条结果后，可以：
- 打开文件
- 定位文件夹
- 复制完整路径
- 复制相对路径
- 加入收藏
- 发送到对比右侧

这部分很适合日常高频使用。

## 8. 最近目录和收藏文件怎么试

### 最近目录

### 收藏文件

如果你是试用者，最值得反馈这几类问题：

- 第一次打开时，能不能快速明白怎么用

- Help 文案是否足够直白

- 搜索文件是否快
- 打开 / 定位 / 复制路径是否顺手
- 最近目录 / 收藏文件是否有价值

### 9.3 对比能力
- 文件 diff 是否清楚
- 文件夹差异是否好理解
- AI 总结是否有帮助

### 9.4 稳定性
- 是否有按钮点了没反应
- 是否有窗口排版问题
- 是否出现异常报错
- 是否有路径 / 文件打不开的情况

## 10. 常见问题

### Q1：为什么搜索不到文件？
通常检查三件事：
- 目标文件夹是否选对
- 关键词是否准确
- 忽略规则是否把目标文件过滤了

### Q2：为什么对比后“多余项”很多？
说明真实目录里有不少内容没有写进模板。
这不一定是错，也可能是：
- 模板太旧
- 项目后期新增了内容
- 中间文件 / 临时文件没有被忽略

### Q3：一键补齐为什么没有补我想补的东西？
因为它只能补模板中**已经定义**的内容。
模板没写的，它不会自动猜。

### Q4：左右到底谁是 old，谁是 new？
记住：
- 左 = old
- 右 = new

### Q5：这个工具会不会直接改我很多文件？
只有在你主动点击“生成结构”或“一键补齐缺失项”时，它才会创建缺失目录或空文件。
普通搜索、扫描、对比不会大规模修改你的文件。

## 11. 推荐的试用任务

如果你想快速判断这个工具值不值得继续做，建议拿下面这几类任务来测：

### 场景 A：历史项目整理
找一个已经有点乱的老项目目录，测试：
- 反向生成模板
- 扫描结构
- 对比差异

### 场景 B：新项目建标准结构
找一个空目录，测试：
- 加载模板
- 初始化模板
- 生成结构
- 保存项目配置

### 场景 C：文档版本对比
拿两个相近版本的文本文件，测试：
- 左右文件对比
- diff 高亮
- AI 总结差异

### 场景 D：高频文件管理
找一个资料型项目，测试：
- 搜索文件
- 收藏常用文件
- 复制相对路径
- 打开最近目录

## 12. 一句话总结

这不是一个简单的“建文件夹工具”。
它更像一个面向项目资料治理的本地工作台：

- 先定规则
- 再建结构
- 再查偏差
- 再提升找文件和交接效率

如果你在试用过程中，能明显感受到：
- 项目目录更清楚了
- 文件更好找了
- 差异更容易说明白了

那就说明这个方向已经开始接近产品了。
