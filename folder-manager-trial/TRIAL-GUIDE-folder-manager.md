# 哈利 / Windows 文件夹管理系统试用说明

这是一份给客户、同事、试用者直接看的简版说明。

## 这是什么

这是一个本地桌面工具，用来管理项目文件夹结构。

它能帮你做几件事：
- 按模板一键生成标准目录
- 扫描当前目录结构
- 对比“标准结构”和“真实结构”差异
- 一键补齐缺失目录或文件
- 搜索、打开、定位、复制文件路径
- 保存项目配置，减少重复设置
- 收藏常用文件，快速进入高频资料

## 适合谁用

适合这些人：
- 需要整理项目资料的人
- 需要建立项目目录规范的人
- 经常找文件、交接文件的人
- 需要管理研发、交付、归档资料的人

## 你会得到什么

使用后，最直接的变化通常是：
- 文件更好找
- 目录更统一
- 新人更容易接手
- 项目归档更省事
- 少掉很多“这个文件在哪”的沟通成本

## 最简单的使用方式

第一次使用，建议按这个顺序：

1. 选择一个项目文件夹
2. 选择模板类型并点击“加载模板”
3. 点击“初始化模板”
4. 点击“生成结构”
5. 点击“扫描结构”
6. 打开“文件/文件夹对比”查看左右差异
7. 需要时点击“一键补齐缺失项”
8. 在右侧搜索文件、打开文件、复制路径

## 对比窗口怎么看

记一个规则就够了：
- 左边 = OLD / 基准
- 右边 = NEW / 当前

颜色说明：
- 绿色：新增
- 红色：删除
- 蓝色：文件头 / 路径信息
- 紫色：变更块

如果你只想快速看结论：
- 先看右侧 diff
- 再看下方 AI 总结

## 新版增强点

当前试用版已经支持：
- 最近打开目录
- 收藏常用文件
- 项目配置保存 / 自动加载
- 文件 / 文件夹左右对比
- 差异高亮显示
- 复制相对路径
- 精简 Help 说明
- 关于窗口和版本号展示

## 适合先拿什么项目试

建议先拿下面这类项目试：
- 一个中小型历史项目目录
- 文件层级已经有点乱的项目
- 需要整理归档的资料包
- 需要多人接手的项目

## 试用建议

- 第一次先不要直接上正式核心项目
- 先用一个测试目录或副本练手
- 如果你已经有历史项目，也可以先用“反向生成模板”看看现状
- 常用文件建议加入收藏
- 固定项目建议保存项目配置

## 一句话总结

这不是单纯“建文件夹”的工具，
它更像一个让项目资料更有秩序、更容易查找、更方便交接的本地管理台。
