# 哈利 / Windows 文件夹管理系统原型说明

这是一个 Python 原型，用来验证这套“类似 GitHub 的本地文件夹管理系统”思路。

## 当前功能

1. 根据 `File_list.md` 一键创建目录和空文件
2. 扫描目标目录，把真实结构导出到 Markdown 文件
3. 对比 `File_list.md` 与真实目录结构，输出差异日志
4. 提供多模板导入：通用模板、DHF模板、硬件模板、结构模板
5. 提供 `Help` 按钮，内置精简教程
6. 支持忽略规则，可跳过 `.git`、`node_modules` 等目录
7. 支持从当前真实目录 **反向生成** `File_list.md`
8. 支持 **一键补齐缺失项**
9. 支持文件搜索、打开文件、定位文件夹、复制完整路径 / 相对路径
10. 支持项目配置保存 / 自动加载
11. 支持最近目录、收藏文件
12. 支持文件 / 文件夹左右对比
13. 对比窗口明确区分 `左侧（OLD / 基准）` 与 `右侧（NEW / 当前）`
14. diff 高亮说明已内置：绿色新增、红色删除、蓝色文件头、紫色变更块
15. 支持 AI 差异总结、结构化结果查看、版本号与关于窗口

## 主程序

- 文件：`file-folder-manager.py`
- 运行环境：Python 3.10+
- GUI：`tkinter + ttk`

## 界面布局

### 一级界面 1：项目设置

包含：

- 目标文件夹选择
- 模板选择与加载
- Help 按钮
- 忽略规则输入
- 结构操作按钮组
- `File_list.md` 编辑区

### 一级界面 2：文件检索与对比

包含：

- 文件搜索区
- 打开 / 定位 / 复制路径操作按钮
- 搜索结果表格
- 运行日志
- 文件 / 文件夹对比弹窗
- 结构化结果窗口

对比窗口的阅读规则：
- 左侧 = `OLD / 基准`
- 右侧 = `NEW / 当前`
- 绿色 = 新增
- 红色 = 删除
- 蓝色 = 文件头 / 路径信息
- 紫色 = 变更块定位

这种布局比之前更适合长期使用：

- 左边负责“搭结构、管规则”
- 右边负责“找文件、看结果”

## 如何运行

```bash
python file-folder-manager.py
```

Windows 上也可以：

```bash
py file-folder-manager.py
```

## 建议使用流程

最短路径：
1. 选择目标文件夹
2. 加载模板
3. 初始化模板
4. 生成结构
5. 扫描结构
6. 打开“文件/文件夹对比”查看差异
7. 需要时再做 AI 总结


### 1）准备目标文件夹

先新建一个空目录，例如：

- `D:\ProjectManagerDemo`

### 2）启动程序并选择目标文件夹

点击“选择文件夹”。

### 3）选择模板类型

程序已内置 4 种模板：

- `通用模板`
- `DHF模板`
- `硬件模板`
- `结构模板`

选择后点击“加载模板”，编辑区会自动切换到对应初始化内容。

### 4）设置忽略规则

在“忽略目录/文件”输入框中填写要跳过的名称，多个用英文逗号分隔。

默认值：

- `.git,node_modules,__pycache__,.idea,.vscode,dist,build`

这些规则会影响：

- 目录扫描
- 差异比对
- 反向生成模板
- 文件搜索

### 5）初始化模板

点击“初始化模板”，会在目标目录下生成：

- `File_list.md`

### 6）编辑目录模板

你可以直接在程序里的编辑框修改，也可以打开目标目录下的 `File_list.md` 修改。

格式规则：

- 每层缩进 2 个空格
- 目录以 `/` 结尾
- 文件直接写文件名
- `#` 后面可写说明注释

### 7）生成结构

点击“生成结构”，程序会：

- 创建缺失目录
- 创建缺失文件
- 保存当前编辑区内容到目标目录下的 `File_list.md`

### 8）扫描结构到 Markdown

点击后会在目标目录下创建：

- `snapshots/tree-时间戳.md`

### 9）对比差异并记录日志

点击后会在目标目录下创建：

- `logs/compare-时间戳.log`

同时，界面右下方“差异结果”区域会直接显示：

- 缺失项
- 多余项

### 10）一键补齐缺失项

点击后，程序会把当前差异中的缺失目录/文件自动补齐。

### 11）双击差异项定位

在“差异结果”区域双击某一条路径，程序会尝试在系统文件管理器中打开对应位置。

### 12）文件搜索与快速拿取

在右侧“文件搜索”区域输入关键词后点击“搜索文件”，程序会列出匹配结果，包含：

- 文件名
- 相对路径
- 文件大小
- 修改时间

搜索结果支持：

- 同时匹配 **文件名** 和 **相对路径**
- 支持多个关键词按空格分隔进行联合搜索
- `显示全部文件`：不输入关键词也能直接列出当前目录下全部文件
- `打开文件`：直接用系统默认程序打开
- `定位文件夹`：打开该文件所在目录
- `复制完整路径`：一键复制路径，便于发给别人或粘贴到其他软件
- 双击搜索结果：直接打开文件
- 搜索日志会显示：当前搜索目录、已扫描文件数、关键词和命中数量

### 13）从当前目录反向生成模板

点击后，程序会根据当前真实目录结构自动生成一份新的 `File_list` 内容，直接回填到编辑区，方便你：

- 快速建立标准模板
- 从历史项目反推目录规范
- 先扫描现状，再做结构治理

## 模板说明

## 打包 EXE

已提供一键打包脚本：

- `build-folder-manager-exe.bat`

在 Windows 电脑上把整个项目文件夹拷过去后，双击这个 `.bat` 即可。它会自动：

1. 创建独立虚拟环境 `.venv-build`
2. 安装 `PyInstaller`
3. 打包 `file-folder-manager.py`
4. 输出成单文件 EXE

默认输出路径：

- `dist/Windows文件夹管理系统.exe`

如果你想手动打包，也可以在 Windows 命令行里执行：

```bash
py -m venv .venv-build
.venv-build\Scripts\activate
python -m pip install --upgrade pip
pip install pyinstaller
pyinstaller --noconfirm --clean --windowed --onefile --name "Windows文件夹管理系统" file-folder-manager.py
```

注意：

- 由于这是 `tkinter` 桌面程序，建议在 **Windows 本机** 打包 Windows `.exe`
- 我当前这里是 Linux 环境，不能直接产出可用的 Windows `.exe`
- 如果后面你要加图标，可以再补 `--icon your.ico`

如果你要把它变成双击可用的软件，后续可以用：

```bash
pip install pyinstaller
pyinstaller --onefile --windowed file-folder-manager.py
```

打包后一般会在：

- `dist/file-folder-manager.exe`
